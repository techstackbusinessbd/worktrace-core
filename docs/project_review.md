# WorkTrace - Full Project Review Report

এই রিপোর্টটিতে WorkTrace પ્રોজেক্টের বর্তমান টেকনিক্যাল অবস্থা, সম্পূর্ণ হওয়া ফিচারসমূহ এবং ভবিষ্যৎ কাজের একটি পূর্ণাঙ্গ রিভিউ তুলে ধরা হলো।

## 🏗️ Technical Architecture

### Backend (Laravel 11 API)
- **Framework:** Laravel 11 (API Only).
- **Database:** PostgreSQL (Dockerized).
- **Authentication:** Laravel Sanctum. ড্যাশবোর্ড ইউজারদের জন্য পার্সোনাল অ্যাক্সেস টোকেন এবং ডেক্সটপ এজেন্টদের জন্য ডিভাইস-স্পেসিফিক টোকেন ব্যবহৃত হচ্ছে।
- **Multi-Tenancy:** Custom Tenant ID based scoping. (প্রতিটি মডেলে `tenant_id` দিয়ে ডাটা আলাদা করা হয়েছে, যাতে এক কোম্পানির ডাটা অন্য কোম্পানি দেখতে না পায়)।
- **Storage:** লোকাল স্টোরেজ (ভবিষ্যতে S3 বা MinIO-তে শিফট করার অপশন রাখা আছে)।

### Frontend (React + Vite)
- **Framework:** React 19 + Vite.
- **State Management:** Zustand (ইউজার সেশন এবং অথেনটিকেশন স্টেট মেইনটেইন করার জন্য)।
- **UI & Styling:** Custom Vanilla CSS (Tailwind ছাড়া) এবং "Material You" ইনস্পায়ার্ড ডিজাইন সিস্টেম।
- **Icons:** Lucide React.
- **Routing:** React Router v7.

## ✅ Completed Features (হয়ে যাওয়া কাজগুলো)

### ১. Authentication & Onboarding
- **Company Registration:** নতুন কোম্পানি সাইন-আপ করতে পারে।
- **Superadmin vs Admin:** সিস্টেমের সুপারঅ্যাডমিন এবং নির্দিষ্ট কোম্পানির (Tenant) অ্যাডমিন রোল তৈরি করা হয়েছে।
- **Login Flow:** সিকিউর লগইন এবং ড্যাশবোর্ড রিডাইরেকশন।

### ২. Agent Enrollment & Tracking API
- **Silent Enrollment:** এজেন্ট সফটওয়্যারটি ইমেইল/পাসওয়ার্ড ছাড়াই শুধুমাত্র "Enrollment Token" ব্যবহার করে ব্যাকএন্ডে কানেক্ট হতে পারে।
- **Telemetry Endpoints:** এজেন্ট থেকে প্রতি ১০ মিনিট পরপর ডাটা (অ্যাক্টিভ টাইম, মাউস ক্লিক, কীবোর্ড স্ট্রোক, অ্যাপ্লিকেশনের নাম) এবং স্ক্রিনশট রিসিভ করার জন্য API তৈরি করা হয়েছে।

### ৩. Dashboard Features
- **Devices Management:** কানেক্টেড হওয়া সবগুলো কম্পিউটার দেখার পেজ।
- **Bulk Mapping & Auto-Assign:** এন্টারপ্রাইজ লেভেলে একসাথে একাধিক ডিভাইসকে "Smart Match" বা "Auto-Create" দিয়ে এমপ্লয়িদের সাথে লিংক করার দারুণ ফিচার।
- **Timesheets (Basic):** নির্দিষ্ট এমপ্লয়ির প্রতিদিনের কাজ করা সময়, কোন অ্যাপে কতক্ষণ ছিল, সেগুলোর সামারি দেখার পেজ।

## 🛠️ Known Issues & Limitations (যেসব সীমাবদ্ধতা আছে)

> [!WARNING]
> ১. **Screenshots Viewer:** টাইমশিট পেজে স্ক্রিনশটগুলোর নাম বা ডাটা আসছে ঠিকই, কিন্তু সেগুলো সুন্দর গ্যালারি বা লাইটবক্স আকারে দেখার UI এখনো বানানো হয়নি।
> ২. **Dashboard Overview:** মেইন ড্যাশবোর্ড পেজটি (যেখানে কোম্পানির টোটাল কাজের গ্রাফ বা চার্ট থাকার কথা) এখনো ফাঁকা।
> ৩. **Role Permissions:** এমপ্লয়িরা লগইন করলে তারা কী কী দেখতে পারবে, সেই পারমিশন লজিক ফ্রন্টএন্ডে পুরোপুরি সেটআপ করা হয়নি (বর্তমানে অ্যাডমিন ফোকাসড কাজ হয়েছে)।
> ৪. **Timezone Adjustments:** ট্র্যাকিং ডাটা UTC ফরমেটে সেভ হচ্ছে, ড্যাশবোর্ডে দেখানোর সময় ইউজারদের লোকাল টাইমজোনে কনভার্ট করার কাজ বাকি।

## 🚀 Next Steps (পরবর্তী করণীয়)

প্রোজেক্টটিকে পূর্ণাঙ্গ রূপ দেওয়ার জন্য আমাদের নিচের কাজগুলো করতে হবে:

1. **Dashboard Analytics (Overview Page):**
   - আজকে কতজন এমপ্লয়ি অনলাইনে আছে।
   - টোটাল কত ঘন্টা কাজ হয়েছে তার একটি বার-চার্ট বা পাই-চার্ট তৈরি করা।
2. **Screenshot Gallery:**
   - টাইমশিটে স্ক্রিনশটগুলো গ্রিড আকারে সাজানো।
   - ক্লিক করলে বড় করে দেখার জন্য Image Modal (Lightbox) যুক্ত করা।
3. **Employees Page Management:**
   - ড্যাশবোর্ড থেকে নতুন এমপ্লয়ি ইনভাইট করা বা অ্যাড করার পেজ ডিজাইন করা।
4. **Billing & Subscription:**
   - কোম্পানির ট্রায়াল পিরিয়ড শেষ হলে পেমেন্ট গেটওয়ে বা প্ল্যান আপগ্রেড করার সিস্টেম তৈরি করা।
5. **Agent Software Development:**
   - এতক্ষণ আমরা `simulate-agent.js` দিয়ে ব্যাকএন্ড টেস্ট করেছি। এবার আসল C# বা Electron.js দিয়ে ডেক্সটপ এজেন্ট বানানোর কাজ শুরু করতে হবে।
